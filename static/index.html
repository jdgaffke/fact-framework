SOA Composition Service (SCS)

What is it?

  • An service for executing data operators, where operators are either local (in-process) or remote (out-of-process, over the network) services.
  • Usage:
    1. A service client submits a query in XML format. The query describes the operators to invoke, and how the output of one operator feeds into the input of the next.
    2. The Composition Service executes the operators in the optimal sequence, formatting the results into a XML document.

Benefits to SOA clients:

  • Provides a more declarative SOA approach.
  • Frees SOA clients from concerns of orchestrating multiple service calls, stitching together results, parallelization, etc.
  • Simplified data exchange: XML query input, XML recordset output. XML allows XSD-based tools.
  • Operator metadata is self-documenting, always in sync, supports searchable documentation
  • Browser-based REPL simplifies development

Benefits to SOA developers:

  • Service developers focus on implementing data operators only. Execution engine handles operator orchestration including sequencing, parallelization, data caching, etc.
  • Metadata-driven access to execution engine services, such as automatic data caching, data security, provisioning.
  • Handles versioning, deprecations, retirement of operators
  • Less concern about API boundaries = faster development
  • Execution plan tracing, statistics, resource governance.
